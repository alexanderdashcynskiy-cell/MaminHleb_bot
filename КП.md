# Коммерческое предложение

## Разработка цифровой экосистемы для пекарни «Мамин Хлеб»

**Кому:** руководству пекарни «Мамин Хлеб»
**Дата:** 25 июня 2026 г.
**Срок действия предложения:** 30 календарных дней

---

## 1. О предложении

Предлагаем комплексную разработку и сопровождение двух взаимосвязанных
продуктов, образующих единую цифровую экосистему пекарни:

| № | Продукт | Назначение | Аудитория |
|---|---------|------------|-----------|
| 1 | **CRM-панель «Мамин Хлеб»** | Управление заказами, складом, кассой, персоналом, аналитикой | Владелец, повар, кассир, пекарь |
| 2 | **Telegram-бот + Mini App** | Витрина, приём заказов и предзаказов, программа лояльности | Клиенты пекарни |

Оба продукта работают на единой базе данных и обмениваются событиями в
реальном времени: заказ из Mini App моментально попадает в CRM, статус из CRM —
обратно клиенту в Telegram.

## 2. Проект №1 — CRM-панель управления пекарней

### 2.1 Краткое описание

Внутренняя панель управления для сотрудников пекарни. Single-Page Application
на React 19 с серверным API на Express и базой PostgreSQL. Реализована система
ролей, аналитика, голосовой и AI-ассистент.

### 2.2 Функциональные модули

- **Дашборд и аналитика** — выручка, число заказов, средний чек, конверсия,
  почасовая динамика, топ-товары и топ-продавцы. Интерактивные графики (Recharts).
- **Заказы** — создание, редактирование, смена статуса, отмена, запрос отзыва у
  клиента, ответы на отзывы. Поддержка предзаказов.
- **Касса** — оформление продаж, выбор количества и места, способы оплаты,
  фиксация расхождений.
- **Склад / Остатки** — добавление, редактирование и удаление товаров, загрузка
  изображений, отметки «Новинка» / «Хит». 7 категорий: Хлеб, Торты, Десерты,
  Пироги, Пирожки, Пицца, Кофе.
- **Персонал** — учёт сотрудников, роли, смены (журнал входов/выходов),
  отчёты пекаря.
- **Отзывы и комментарии** — модерация, ответы клиентам через Telegram.
- **Системный раздел** — настройки, webhook Telegram, «счастливый час»,
  служебная статистика.
- **AI-ассистент (Google Gemini)** — генерация отчётов, ответов и аналитических
  сводок; голосовой ввод.
- **Ежедневные отчёты** — автозакрытие смены, архив отчётов, выгрузка в PDF и
  отправка администратору в Telegram.

### 2.3 Роли и права доступа

| Роль | Доступ |
|------|--------|
| `chef` (повар/владелец) | Полный доступ ко всем модулям |
| `cashier` (кассир) | Заказы и склад |
| `baker` (пекарь) | Только остатки склада |

### 2.4 Технологический стек

- **Frontend:** React 19, TypeScript, Vite 6, Tailwind CSS, Recharts, Motion
- **Backend:** Node.js, Express, TypeScript
- **База данных:** PostgreSQL + Prisma ORM (модели: Order, Product, DailyReport,
  Setting, Employee, SessionLog)
- **Интеграции:** Telegram Bot API (webhook), Google Gemini AI
- **Безопасность:** JWT (httpOnly-cookie), bcrypt, Helmet, rate-limiting

### 2.5 Объём кода

- `src/App.tsx` — ~6 100 строк (основной фронтенд)
- `server.ts` — ~1 950 строк (API, 40+ эндпоинтов)
- 25+ React-компонентов (вкладки, модальные окна, элементы UI)
- Покрытие тестами: Vitest (unit) + Playwright (e2e)

## 3. Проект №2 — Telegram-бот и Mini App

### 3.1 Краткое описание

Клиентский продукт: Telegram-бот с полноценным интернет-магазином внутри
Mini App. Витрина с каталогом, корзина, оформление заказов и предзаказов,
программа лояльности и отзывы.

### 3.2 Функциональные модули

- **Витрина / каталог** — карусели акций (бренд, «счастливый час»,
  предзаказ), категории, карточки товаров с фото, индикатор остатков
  («Осталось N шт.»), хиты и бестселлеры.
- **Корзина и оформление** — добавление товаров, изменение количества,
  ввод данных (телефон, имя, адрес, комментарий), серверный расчёт цены.
- **Предзаказы** — отдельный сценарий с собственной корзиной и формой.
- **«Счастливый час»** — автоматическая скидка 30% по расписанию (отдельно для
  будней и выходных), push-уведомления клиентам, серверный расчёт скидки.
- **Программа лояльности / рефералы** — реферальный код, список приглашённых,
  экран бонусов.
- **Отзывы** — приём оценок и текстовых отзывов, передача в CRM.
- **История заказов** — список прошлых заказов клиента.
- **Уведомления** — статусы заказов клиенту, новые заказы — в админ-чат.

### 3.3 Технологический стек

- **Backend:** Node.js, Express, PostgreSQL (через `pg`, без ORM)
- **Frontend (Mini App):** одностраничное приложение `index.html`, Tailwind CSS,
  самохостинг шрифтов (Montserrat, Playfair Display, Poppins)
- **Интеграции:** Telegram Bot API + Telegram Web App SDK
- **Безопасность:** HMAC-SHA256 верификация `initData`, проверка
  `X-Telegram-Bot-Api-Secret-Token`, ограничение CORS, защита админ-эндпоинтов

### 3.4 Объём кода

- `bot/index.js` — ~1 200 строк (бэкенд: 12+ эндпоинтов, webhook, лояльность)
- `bot/index.html` — ~4 500 строк (Mini App, единственный файл фронтенда)
- 40+ оптимизированных изображений товаров

## 4. Выполненный security-аудит (включён в стоимость)

Оба проекта прошли глубокий аудит безопасности и качества кода (версия 2.2).
Выявлено и устранено **204 проблемы**. Ключевые исправления:

**CRM:**
- Удалён `GEMINI_API_KEY` из клиентского бандла (утечка ключа)
- Реализована проверка паролей (bcrypt + миграция legacy plain-text)
- Полный отказ от Google Sheets в пользу PostgreSQL/Prisma
- Корректная отдача исторических отчётов через API

**Bot:**
- Исправлен небезопасный `mode: 'no-cors'` → стандартный fetch
- Ограничен `Access-Control-Allow-Origin` (был `*`)
- Добавлена проверка секрета на `/webhook`
- `state.json` на эфемерной ФС заменён на таблицу PostgreSQL `bot_state` (TTL)
- Верификация `initData` вместо доверия `initDataUnsafe.user.id`
- Ограничение длины полей заказа, экранирование HTML

Сопроводительные документы аудита: `AUDIT.md`, `DEEP-AUDIT-REPORT.md`,
`DATA-SECURITY.md`, `RECONCILIATION-280.md`, `audit-report.pdf`.
